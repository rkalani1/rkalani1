About

Practice, evidence, and the tools that close the gap between them.

I build public demo code at the intersection of acute stroke evidence, access modeling, clinical research methods, and reviewable software. My work sits where three pressures converge: clinicians needing fast, defensible evidence review; networks needing rigorous public methods for access modeling; and researchers needing tools that do not get in the way.

Everything published here is meant to be opened, reviewed, and forked as public demo code. The constraint that makes the tools teachable - static HTML, no backend, no analytics, no PHI - also makes them easy to audit before any approved deployment.

"If a tool cannot be opened and inspected in a single browser tab, it is not ready to be adapted responsibly."

How these tools are built

Four principles, applied across every release.

These aren't aspirations — they're the design constraints I work backward from. If a feature can't be shipped without violating one of them, it doesn't ship.

01

Local-first, no PHI

Data stays in the browser for demos, but public GitHub Pages is not approved storage. No patient identifiers, learner records, restricted data, or confidential data should be entered.

02

Evidence-anchored

Every clinical recommendation traces to a primary citation (PMID / DOI / NCT). Trial eligibility logic is unit-tested. Drift is flagged.

03

Review-grade speed

Mobile-first, static, and easy to inspect. Real operational use belongs in an approved hosting, storage, and governance path.

04

Open and forkable

MIT-licensed. Static HTML. Zero proprietary dependencies. Any clinician or researcher can read the code, fork the repo, and adapt for their own environment.
